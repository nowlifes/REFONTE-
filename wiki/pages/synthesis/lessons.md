## Comment ajouter une leçon

Quand l'utilisateur corrige une erreur ou qu'un bug récurrent est résolu, ajouter en bas :

```
[YYYY-MM-DD] | description de ce qui a mal tourné | règle concrète à suivre dans le futur
```

Exemples de bonnes leçons :
- `[2026-05-05] | Tailwind v4 ne supporte pas les classes custom non définies → toujours utiliser bg-[#hex] explicitement`
- `[2026-05-05] | Modifier raw/ au lieu de pages/ → raw/ est LECTURE SEULE, jamais modifier`
- `[2026-05-05] | Oublier de mettre à jour index.md après ingest → toujours finir par index.md + log.md`
